## Section B — Project Details

### Project Title

**HomeLodge: A Web-Based Homestay Booking Management System with QR Code Door Access and Real-Time Communication**

---

### Problem Background and Proposed Solution

Homestay operators who manage more than one property face additional difficulty when relying on informal reservation channels. Booking requests received through messaging applications or phone calls make it hard to track which unit is available on any given date, prevent double-bookings across properties, and maintain consistent communication with guests. Payment collection is similarly unstructured: there is no systematic way to generate bills, track payment status per property, or apply cancellation policies uniformly. Property access is typically handled through physical key exchanges, which creates scheduling problems when guests check in or out and when housekeeping must occur between bookings. Taken together, these gaps reduce service reliability and increase the administrative workload on the operator as the number of managed properties grows.

HomeLodge is a web-based homestay booking system that addresses these problems by digitalising the reservation process from end to end. The system supports multiple homestay units, each with its own profile, images, pricing, and availability calendar, allowing guests to browse and select a property before booking. Guests can check date availability per unit, submit bookings, and pay through an online payment gateway within a single system. When a booking is submitted, the selected date is held for one day to allow payment. If payment is not received within that window, the booking is cancelled automatically and the date is reopened. Each confirmed booking generates a unique, time-limited QR code that the guest uses to unlock the property door at check-in, removing the need for a physical key handover. The code expires at checkout and can be regenerated by the administrator for housekeeping purposes. The system sends in-app and email notifications for booking confirmations, payment outcomes, and upcoming stay reminders. A real-time chat feature allows guests to contact the administrator directly within the system. Administrators can manage all homestay units, bookings, payments, user accounts, roles, permissions, and system settings through a dedicated interface. All system activity is recorded in an audit log for accountability purposes.

---

### Objectives

The objectives of the project are:

1. To study, collect, and analyse the functional and non-functional requirements for developing a web-based homestay booking management system.
2. To design the system architecture, database schema, and user interface for the HomeLodge system, encompassing all planned modules and two user roles.
3. To develop the HomeLodge web application with eleven (11) functional modules, including homestay management, booking management, payment processing, QR code door access, real-time chat, and role-based access control, following the defined requirements.
4. To test the developed system against the stated functional and non-functional requirements to verify correctness, security, and usability.

---

### Scope

The scope of the system is as follows:

1. The application will be a web-based system accessible via a modern desktop or mobile web browser.
2. The target users of this system are two (2) distinct roles: Guest (User), referring to individuals who book homestay accommodation; and Admin, the homestay operator responsible for managing bookings, payments, access control, and system configuration.
3. The application will support one (1) language: English.
4. The system supports multiple homestay units managed under a single administrator account.
5. This system will consist of eleven (11) modules:
   - 5.1 Authentication Module
   - 5.2 Homestay Management Module
   - 5.3 Booking Module
   - 5.4 Payment Module
   - 5.5 Notification Module
   - 5.6 Chat Module
   - 5.7 User Management Module
   - 5.8 Role and Permission Module
   - 5.9 System Settings Module
   - 5.10 Audit Logs Module
   - 5.11 QR Code Door Access Module
