# CHAPTER 1: INTRODUCTION

## 1.1 Introduction

The homestay industry has grown as an alternative to conventional hotel accommodation, particularly in Malaysia, where operators rent residential properties to guests for short-term stays. The experience tends to be more personal than a commercial hotel, and for operators managing only one property, the booking process is usually manageable through informal channels.

The problem arises when an operator manages more than one unit. Booking requests that arrive through messaging applications or phone calls cannot be tracked systematically across multiple properties. Availability is checked manually, payment is collected without a formal record, and guests access the property by collecting a physical key in person. These are not unusual practices for a small operation, but they do not scale. Errors multiply, records become inconsistent, and the administrative burden grows with every additional unit.

HomeLodge is developed to address this. It is a web-based homestay booking management system that handles the reservation process from end to end, covering availability checking, booking submission, payment collection, door access, and guest communication within a single platform.

---

## 1.2 Problem Background

Homestay operators who manage multiple properties without a formal system face a range of recurring difficulties that become more severe as the number of units and bookings grows. Booking requests received through messaging applications or phone calls are processed manually, with no shared availability record to prevent two guests from reserving the same dates on the same property. When such conflicts arise, one booking must be cancelled. Payment collection is similarly unstructured: operators collect payment by bank transfer or cash, generate receipts by hand, and calculate refunds manually, leaving no consistent record to support dispute resolution. Property access depends on physical key handovers, which require the operator or a representative to be present at each check-in. When an operator manages several properties, coordinating simultaneous check-ins and housekeeping between bookings becomes a practical problem, and physical keys carry the added risk of unauthorised duplication with no audit trail of who accessed the property and when. Guest communication occurs through personal messaging applications, where booking confirmations, payment instructions, and enquiries accumulate in threads that are not linked to any booking record and cannot be reviewed centrally by the operator. Finally, reminders for unpaid bookings and upcoming check-ins are sent manually, which is easy to overlook when multiple properties and reservations are running concurrently, and guests who miss a payment deadline may lose their booking without prior warning.

To address these problems, HomeLodge is developed as a web-based homestay booking management system that digitalises the entire reservation process. Each homestay unit is given its own profile with images, pricing, and an availability calendar so that guests can check dates and submit bookings without contacting the operator directly. When a booking is submitted, the selected dates are held for twenty-four hours; if payment is not received within that period, the booking is cancelled automatically and the dates are released. Payment is processed through an online payment gateway, and billing documents and receipts are generated by the system without manual input. Each confirmed booking produces a unique, time-limited QR code for door access, removing the need for a physical key handover, and the administrator can regenerate a separate code for housekeeping purposes between stays. Guest and administrator communication is handled through a real-time chat interface built into the system, and automated notifications are sent by email and within the application for booking confirmations, payment outcomes, pending payment reminders, and upcoming stay reminders. The administrator manages all units, bookings, payments, users, roles, permissions, and system settings through a single interface, and every system action is recorded in an immutable audit log.

## 1.3 Project aim

The aim of this project is to develop HomeLodge, a web-based homestay booking management system that digitalises the reservation, payment, property access, and communication processes for operators managing multiple homestay units.

---

## 1.4 Project objectives

The objectives of this project are:

1. To study, collect, and analyse the functional and non-functional requirements for a web-based homestay booking management system.
2. To design the system architecture, database schema, and user interface for HomeLodge, covering all planned modules and both user roles.
3. To develop the HomeLodge web application with eleven (11) functional modules, including homestay management, booking, payment, QR code door access, real-time chat, and role-based access control, in accordance with the defined requirements.
4. To test the developed system against the stated functional and non-functional requirements to verify correctness, security, and usability.

---

## 1.5 Project scope

The scope of this project is as follows:

1. The system is a web-based application accessible through a modern desktop or mobile web browser.
2. Two (2) user roles are supported:
   - Guest: an individual who browses, books, and pays for homestay accommodation.
   - Admin: the homestay operator who manages bookings, payments, access control, user accounts, and system configuration.
3. The application operates in English only.
4. Multiple homestay units can be managed under a single administrator account.
5. The system comprises eleven (11) functional modules:
   - 5.1 Authentication Module
   - 5.2 Homestay Management Module
   - 5.3 Booking Module
   - 5.4 Payment Module
   - 5.5 Notification Module
   - 5.6 Chat Module
   - 5.7 User Management Module
   - 5.8 Role and Permission Module
   - 5.9 System Settings Module
   - 5.10 Audit Logs Module
   - 5.11 QR Code Door Access Module
6. The QR code door access module is designed for compatible smart lock devices that support QR code scanning. Physical installation of lock hardware is not part of this project.
7. The payment gateway supports online payment processing. The specific payment provider depends on availability and configuration at the time of deployment.
8. The system does not include multi-language support in this version.

---

## 1.6 Project importance

HomeLodge is developed to address the operational difficulties faced by homestay operators who manage multiple properties without a formal booking system. Managing reservations, payments, and property access through informal channels is error-prone and unsustainable as the number of units grows, and the system is therefore needed to replace these manual processes with a structured, digital alternative. The system benefits homestay operators by automating availability tracking, payment collection, billing, and guest notifications, which reduces the risk of double-bookings and eliminates the need for manual follow-up at every stage of a reservation. Guests benefit from the ability to check availability, submit bookings, and make payment independently without waiting for a manual response, and they can access the property using a time-limited QR code rather than collecting a physical key. The system also benefits both parties through improved accountability: all booking, payment, and access events are logged automatically, role-based access control restricts actions to those each user is authorised to perform, and QR code credentials expire at checkout without any manual intervention. Overall, HomeLodge is expected to reduce the administrative burden on operators managing multiple units while providing guests with a more reliable and self-contained booking experience than informal channels currently allow.
