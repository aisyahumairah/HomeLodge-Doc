# CHAPTER 1: INTRODUCTION  
**Writing Guidelines - PSM 1**

## 1.1 Introduction
- Introduction to the main questions/problems to be solved in this project

## 1.2 Problem Background
Problem background should highlight:
- Detailed issues faced by the user  
- Specific problems (manual activities performed by the user that need to be improved)  

**Proposed Solutions:**
- Suggested improvements to solve the aforementioned issues and problems  
- Potential features to automate the manual/problematic activities  

## 1.3 Project Aim
The main goal to be achieved at the end of the project.  
Each project should have only **one goal**, written in a concise and clear paragraph.

## 1.4 Project Objectives
- Consist of tasks to be done towards achieving the goal  
- Should align with the system development methodology  
- Written in numbered form  
- Clearly defined and measurable (the level of success of each task can be evaluated)  

## 1.5 Project Scope
Describe the scope of the project, including:
- Constraints  
- Boundaries  
- Limitations of the system  

Write in numbered or point form.  
Example: scope for a student management system.

## 1.6 Project Importance
Explain:
- Why the system needs to be developed  
- Benefits to be gained  
- Who will benefit and how  

## 1.7 Report Organization
Explain how the report is structured and organized.
